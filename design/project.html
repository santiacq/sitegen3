dotmgr

github · docs · install

A minimal dotfile manager written in Go. It symlinks your config files from a central repo into the right places, supports basic templating, and does nothing else.

why

I tried GNU Stow, chezmoi, and yadm. They all worked, but each came with tradeoffs I didn't love. Stow doesn't do templates. Chezmoi does too much. Yadm uses a bare git repo which makes me nervous.

I wanted something in between: symlinks plus simple variable substitution, in a single binary with no dependencies.

how it works

You keep your dotfiles in a directory. Each subdirectory maps to a target location. A simple YAML file defines variables and any overrides. 

# dotmgr.yml
target: ~/.config
vars:
  editor: nvim
  theme: gruvbox

ignore:
  - "*.bak"
  - ".DS_Store"

Files ending in .tmpl are processed as Go templates before symlinking. Everything else is linked as-is. 

$ dotmgr link
  ~/.config/nvim/init.lua → dotfiles/nvim/init.lua
  ~/.config/alacritty/alacritty.toml → dotfiles/alacritty/alacritty.toml.tmpl
  ~/.config/git/config → dotfiles/git/config
  3 files linked, 0 errors

status

Stable and feature-complete for my needs. I use it daily across two machines. PRs welcome if you find a bug, but I'm not planning to add major features.
